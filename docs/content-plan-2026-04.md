## 1. למה התוכנית הזו

המיפוי הנוכחי מצביע על שני פערים:

**פער 1 — הבלוג לא פונה לרופאים.** קיימים 6 פוסטי בלוג בסך הכל, כולם כתובים לקהל מפתחים (תבניות system prompt של "senior developer", "DevOps expert", פוסט על MCP revolution ברמה טכנית גבוהה). אין אפילו פוסט אחד שממחיש שימוש רפואי.

**פער 2 — חידושי Claude מהרבעון האחרון לא מיוצגים באתר.** בין ינואר לאפריל 2026 שוחררו: Claude Desktop native app, Cowork (task orchestrator חדש), Opus 4.7 עם חלון הקשר של מיליון טוקנים, ושדרוגים משמעותיים ל-Claude Code (hooks, PowerShell, MCP OAuth). אף אחד מהם לא מוזכר בקורסים הקיימים או בבלוג.

**יעד התוכנית:** לסגור את שני הפערים בעזרת 13 יחידות תוכן שכולן ממוקדות רופא, מבוססות use case, וכוללות audit רפואי ו-audit סגנוני לפני פרסום.

### 2.2 רמת קורא

הפרויקט משתמש במערכת רמות עם markers inline:

```
:::beginner
הסבר פשוט למתחיל
:::

:::advanced
פרטים טכניים למתקדם
:::
```

**בעיה שזוהתה:** גוף הטקסט הראשי לעתים כבד מדי לרופא לא-טכני. דוגמה מ-`what-is-claude`: המונחים "Constitutional AI" ו-"RLHF" מופיעים בגוף הטקסט בלי הסבר, ורק ב-`:::advanced` יש פירוט. הרופא שאינו מפתח נתקע באמצע.

**כלל תיקון לכל תוכן חדש:** הגוף הראשי כתוב לרמת "רופא-לא-מפתח". `:::beginner` רק מחזק, `:::advanced` רק מעמיק. אף מונח טכני לא מופיע בגוף הראשי בלי הסבר חד-משפטי מיד אחריו.

### 2.3 שפה ותרגום

בדיקה מדגמית של `what-is-claude` ו-`mcp-revolution`:
- העברית טבעית ונקייה (לא Google Translate).
- אבל יש **רצפי אנגלית ארוכים** (`Claude Code`, `MCP`, `Model Context Protocol`, `Agent SDK`) באמצע משפטים עבריים — זה שובר RTL ב-mobile.
- **מספרים עירומים באמצע משפט עברי** ("200,000 טוקנים", "4.5") — לפעמים ה-RTL נשבר.
- **אין תעתיק עברי** ל-`Claude`, `Claude Code`, `Cowork`. כולם נשארים באנגלית.

### 2.4 סימני AI בכתיבה הקיימת

סריקה מצאה הופעות של: `comprehensive`, "מקיף ו...", משפטי סיכום עם "בסופו של דבר", ו-em dashes (`—`) בעברית.

## 3. החלטות תוכן מנחות

**קהל יעד מדויק:** רופא עברי-דובר (או דו-לשוני), לא מפתח, עובד במרפאה או בית חולים בישראל, בעל היכרות כללית עם ChatGPT אבל לא עם Claude. בעל הבנה טכנולוגית בסיסית (יודע לפתוח אפליקציה, להעלות PDF, להשתמש בדפדפן). לא יודע לקרוא קוד.

**3 עקרונות כתיבה לכל יחידת תוכן:**

1. **כל שיעור חייב לפתוח בשאלה רפואית** ("האם נתקלת אי פעם ב-...?") לפני כל הסבר טכני.
2. **כל תכונה טכנית תמחיש use case רפואי בתוך 3 פסקאות** — לא "מה זה", אלא "מה זה עושה בשבילך כרופא".
3. **כל טענה קלינית עם מקור** — PubMed / UpToDate / AAD / EADV / משרד הבריאות. מובהר מקור בסוף הפסקה או בסוף השיעור.

### קבוצה 1 — פוסטי בלוג חדשים (עדיפות גבוהה ביותר)

**יחידה 1.1 — `desktop-for-physicians`**
- **קטגוריה:** updates
- **כותרת (HE):** "Claude Desktop לרופא — המדריך המלא"
- **כותרת (EN):** "Claude Desktop for Physicians — The Complete Guide"
- **readTime:** 7
- **מטרה:** להכיר את האפליקציה Native של Claude למק/חלונות עם דגש על מה רופא מרוויח ממנה.
- **Features מאומתים לכיסוי:** parallel sessions (כמה שיחות במקביל), computer use, visual diff, Git worktrees, connectors (Google Drive, Slack, Notion), Dispatch בטאב Cowork, scheduled tasks מקומיים.
- **Use cases לכיסוי:** (1) סקירת PDF פתולוגיה בחלון נפרד במקביל לכתיבת מכתב הפניה; (2) connector ל-Google Drive לתיקיות מטופלים; (3) משימה מתוזמנת מקומית ("כל בוקר סרוק את תיקיית ה-PDF והוצא רשימה"); (4) computer use לקריאת דוח בקובץ שסגור בו עותק בפורמט ישן.
- **מקורות:** docs.claude.com/desktop, support.anthropic.com
- **אל תצטט:** "Phone Dispatch" (השם שגוי — הפיצ'ר הנכון הוא "Dispatch" בטאב Cowork).

**יחידה 1.2 — `cowork-for-clinics`**
- **קטגוריה:** updates
- **כותרת (HE):** "Cowork לרופא — לעבוד עם סוכן במקום לחכות לו"
- **כותרת (EN):** "Cowork for Physicians — Working With an Agent, Not Waiting For One"
- **readTime:** 8
- **מטרה:** להסביר את Cowork כטאב צ'אט בו Claude מטפל במשימות רקע תוך כדי שהרופא ממשיך לעבוד. **חשוב:** להבחין ברור בין **Cowork** (טאב צ'אט), **Dispatch** (מכניקת הפעלת משימה בתוך Cowork), **Routines** (משימות חוזרות מבוססות ענן), ו-**Desktop scheduled tasks** (משימות מקומיות באפליקציה).
- **Use cases לכיסוי:** (1) Dispatch — "תקרא את חמשת ה-PDFים האלה ותכתוב סיכום journal club"; (2) Routine יומית — "כל יום ב-8:00 סרוק דוא"ל ואסוף דוחות פתולוגיה חדשים"; (3) onboarding מתמחה חדש — סדרת Dispatch משולבים; (4) ניהול רשימת מטופלי follow-up שמתחייבים לחזור — Routine שבועית שמוציאה רשימה.
- **מקורות:** support.claude.com (Cowork + Routines), docs.claude.com
- **אל תצטט:** "Cowork scheduled tasks" (לא קיים בשם הזה). השתמש ב-"Routines" או "Desktop scheduled tasks" לפי ההקשר.

**יחידה 1.3 — `claude-code-for-non-developers`**
- **קטגוריה:** five-min
- **כותרת (HE):** "Claude Code לרופא שלא יודע לכתוב קוד"
- **כותרת (EN):** "Claude Code for Physicians Who Don't Code"
- **readTime:** 6
- **מטרה:** לפרק את המיתוס ש-Claude Code הוא רק למפתחים. להדגים 3 תסריטים no-code + להציג 1M context ו-Opus 4.7 xhigh ככלי רפואי.
- **Features מאומתים להזכרה:** Opus 4.7 עם effort level `xhigh`, 1M token context (ב-Max/Team/Enterprise), PowerShell on Windows, `/ultraplan` ו-`/ultrareview`, hooks.
- **Use cases לכיסוי:** (1) "קח את כל ה-PDFים בתיקייה הזאת ושים את הטקסט בטבלה"; (2) "תמצת את כל הדוחות הפתולוגיים של החודש — הקשר גדול של 1M מאפשר לעבד הכל בשיחה אחת"; (3) "תסדר את התמונות הדרמוסקופיה לפי תאריך ומטופל".
- **הערה:** להדגיש Claude Desktop כ-GUI wrapper ל-Claude Code — זה הנקודת כניסה הטבעית לרופא לא-טכני.

## 8. סיכונים וקווי גיבוי

**סיכון 1 — שינוי במוצרים של Anthropic באמצע הכתיבה.**
- גיבוי: כל יחידה עם feature-specific תכלול שורת "נכון ל-2026-04-22, בדקו [link]".

**סיכון 2 — טענה קלינית שלא מצאנו לה מקור מהימן.**
- גיבוי: אם אין PubMed/guideline ברור, לא נכתוב אותה. עדיף פוסט קצר יותר מאשר פוסט לא מדויק.

**סיכון 3 — הקורא לא טכני בכלל (אפילו פחות ממה שהערכנו).**
- גיבוי: כל יחידה עם `:::beginner` פותחת, glossary חד-משפטי בכל מונח טכני, ו-"אם נתקעת" link לוידאו הדרכה.

**סיכון 4 — feature שצוטט לא מאומת (Phone Dispatch, /ultrareview וכו').**
- גיבוי: סוכן אימות רץ לפני הכתיבה. כל feature לא-מאומת יוחלף ב-feature מאומת או יוסר.
